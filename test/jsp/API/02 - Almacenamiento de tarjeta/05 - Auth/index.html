1. Autorización con tarjeta almacenada

Una vez que tenga la tarjeta almacenada, puede enviar fácilmente una autorización contra ella. En lugar de los datos recopilados de la tarjeta, simplemente envía el token del cliente (Pagador) y la referencia de la tarjeta (Método de pago), Addon Payments obtiene los datos de la tarjeta almacenados de forma segura de nuestros servidore y crea una autorización que luego enviamos al Emisor. Según las normas de cumplimiento de PCI, el código de seguridad (CVN, CVV) no se puede almacenar. Sin embargo, si en su aplicación o sitio web le brinda al cliente la opción de usar una tarjeta almacenada, puede solicitarle nuevamente su código de seguridad e incluirlo en la solicitud.

1.1 Código de ejemplo
// Importamos las clases necesarias para realizar las peticiones
import java.math.BigDecimal;

import org.junit.Test;

import com.global.api.ServicesContainer;
import com.global.api.paymentMethods.RecurringPaymentMethod;
import com.global.api.entities.exceptions.ApiException;
import com.global.api.serviceConfigs.GatewayConfig;

// Configuramos el cliente y los ajustes de la petición
public class Addon {

private RecurringPaymentMethod paymentMethod;

public Addon() throws ApiException {
GatewayConfig config = new GatewayConfig();
config.setMerchantId("addonjavatest");
config.setAccountId("api");
config.setSharedSecret("secret");
config.setServiceUrl("https://remote.addonpayments.com/remote");
ServicesContainer.configureService(config);

// Referencia del cliente
String customerId = "64f0ecc2-7137-4c70-9a5f-f06be095d833"; // Referencia del cliente 64f0ecc2-7137-4c70-9a5f-f06be095d833

// Referencia del método de pago
String paymentId = "8bd38f27-5258-4538-aa77-bb4f59a9cf47"; // Referencia del método de pago 8bd38f27-5258-4538-aa77-bb4f59a9cf47

// Asoaciamos al objeto paymentMethod las referencias anteriores
paymentMethod = new RecurringPaymentMethod(customerId, paymentId);
}

// Lanzamos la petición al servidor de Addon Payments
@Test
public void Authcard() throws ApiException {

try {
// Procesamos una autorización
Transaction response = paymentMethod.charge(new BigDecimal("19.19"))
.withCurrency("EUR")
.execute();

// Obtenemos la respuesta y la mostramos
String result = "Resultado == " + response.getResponseCode(); // 00 == Success
String message = "Mensaje == " + response.getResponseMessage(); // [ test system ] AUTHORISED
String orderId = "Order ID == " + response.getOrderId(); // ezJDQjhENTZBLTdCNzNDQw
String authCode = "Código de autorización == " + response.getAuthorizationCode(); // Código de autorización
String paymentsReference = "Transaction ID (PasRef) == " + response.getTransactionId(); // Identificador de transacción (PASREF)
String schemeReferenceData = "Schema Reference == " + response.getSchemeId(); // MMC0F00YE4000000715

} catch (ApiException exc) {
// Añada aquí su tratamiento de errores
String errorMessage = exc.getMessage();
}
}
}

2. Descripciones

2.1 Configuración del terminal

Debemos introducir los datos de configuración que se facilitan en la llamada de cualificación del soporte técnico de Addon Payments.

  • Merchant_ID = Identificador del comercio.
  • SubAccount = Subcuenta faciltiada en la llamada de cualificación (en caso de que se envíe la operación sin subcuenta, se usará la establecida por defecto en la creación de su terminal).
  • Shared Secret = Contraseña de seguridad del terminal.
--> En caso de que le falte algún dato de los mencionados arriba, debe contactar con su soporte técnico de Addon Payments llamando al teléfono 914 353 028 (opción 2) o enviando un email al correo electrónico soporteaddonpayments.com.
GatewayConfig config = new GatewayConfig();
config.setMerchantId("addonjavatest");
config.setAccountId("api");
config.setSharedSecret("secret");
config.setServiceUrl("https://remote.addonpayments.com/remote"); // Se debe emplear una de las siguientes URL's:
// Sandbox URL para el entorno de pruebas - https://remote.sandbox.addonpayments.com/remote.
// Live URL para el entorno de producción - https://remote.addonpayments.com/remote.
ServicesContainer.configureService(config);
2.2 Referencias

Necesitamos asociar la referencia del cliente y la referencia de la tarjeta a la cual deseamos realizar el cargo.

  • Asociamos la referencia del cliente a la variable "customerId":
    • customerId = "64f0ecc2-7137-4c70-9a5f-f06be095d833";

  • Asociamos la referencia del método de pago a la variable "paymentId":
    • paymentId = "8bd38f27-5258-4538-aa77-bb4f59a9cf47";

  • Creamos el objeto "RecurringPaymentMethod" con las referencias que hemos asociado anteriormente:
    • paymentMethod = new RecurringPaymentMethod(customerId, paymentId);
// Referencia del cliente
String customerId = "64f0ecc2-7137-4c70-9a5f-f06be095d833"; // Referencia del cliente 64f0ecc2-7137-4c70-9a5f-f06be095d833

// Referencia del método de pago
String paymentId = "8bd38f27-5258-4538-aa77-bb4f59a9cf47"; // Referencia del método de pago 8bd38f27-5258-4538-aa77-bb4f59a9cf47

// Asoaciamos al objeto paymentMethod las referencias anteriores
paymentMethod = new RecurringPaymentMethod(customerId, paymentId);
2.3 Envío de la petición de cobro

Para enviar la solicitud a los servidores de Addon Payments, debemos emplear el modelo de excepciones "try - catch".

  • Indicamos al objeto "paymentMethod" que vamos a realizar un cargo de 19.19€ Transaction response = paymentMethod.charge(new BigDecimal("19.19"))
    • Para preautorizar una transacción, se debe modificar le parámetro ".charge()" por ".Authorize()".

  • Posteriormente debemos indicar la moneda en la que vamos a realizar el cargo: .withCurrency("EUR")

  • Por último, enviamos la petición: .execute();
// Lanzamos la petición al servidor de Addon Payments
@Test
public void Auth() throws ApiException {

try {
// Procesamos una autorización
Transaction response = paymentMethod.charge(new BigDecimal("19.19"))
.withCurrency("EUR")
.execute();

// Obtenemos la respuesta y la mostramos
String result = "Resultado == " + response.getResponseCode(); // 00 == Success
String message = "Mensaje == " + response.getResponseMessage(); // [ test system ] AUTHORISED
String orderId = "Order ID == " + response.getOrderId(); // ezJDQjhENTZBLTdCNzNDQw
String authCode = "Código de autorización == " + response.getAuthorizationCode(); // Código de autorización
String paymentsReference = "Transaction ID (PasRef) == " + response.getTransactionId(); // Identificador de transacción (PASREF)
String schemeReferenceData = "Schema Reference == " + response.getSchemeId(); // MMC0F00YE4000000715

} catch (ApiException exc) {
// Añada aquí su tratamiento de errores
String errorMessage = exc.getMessage();
}
}

Para la respuesta obtenemos en la variable 'errorMessage' el resultado de la función "getMessage()".

Con esta función obtenemos el motivo de la denegación del servidor. Para ver todos los errores de denegación actualizados, puede comprobar nuestra documentación de Addon Payments.
