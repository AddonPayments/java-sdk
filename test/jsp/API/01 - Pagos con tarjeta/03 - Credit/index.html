1. Credit

Este tipo de solicitud le permite ingresar un importe a la tarjeta de un cliente. Para procesar esta solicitud, debe configurar la contraseña de Refund que puede establecer con un miembro de nuestro equipo de soporte Addon Payments. Esta contraseña debe cifrarse utilizando el algoritmo SHA-1 antes de agregarse a la solicitud.

--> Las solicitudes de crédito no se envían al Emisor, se colocan directamente en el archivo de liquidación del día siguiente. Puede contactar con nosotros llamando al teléfono 914 353 028 opción 2 o enviándonos un email a soporte@addonpayments.com.

Recomendamos encarecidamente sanear los datos del titular de la tarjeta antes de enviarla en la solicitud. Por ejemplo, debe comprobar que el número de tarjeta es válido y que la fecha de caducidad es correcta. Nuestra biblioteca JS contiene un conjunto de funciones útiles que le ayudan a procesar las solicitudes correctamente. --> En los ejemplos de esta documentación usamos el fichero de comprobación JS rxp-js.js de RealexPayments. Podrá ver el fichero en la carpeta de "assets/js".

1.1 Código de ejemplo
// Importamos las clases necesarias para realizar las peticiones
import java.math.BigDecimal;

import org.junit.Test;

import com.global.api.ServicesContainer;
import com.global.api.entities.Transaction;
import com.global.api.entities.exceptions.ApiException;
import com.global.api.paymentMethods.CreditCardData;
import com.global.api.serviceConfigs.GatewayConfig;

// Configuramos el cliente y los ajustes de la petición
public class Addon {

private CreditCardData card;

public Addon() throws ApiException {
GatewayConfig config = new GatewayConfig();
config.setMerchantId("addonjavatest");
config.setAccountId("api");
config.setSharedSecret("secret");
config.setRefundPassword("refund");
config.setServiceUrl("https://remote.addonpayments.com/remote");
ServicesContainer.configureService(config);

// Creamos el objeto de la tarjeta
card = new CreditCardData();
card.setNumber("4263970000005262");
card.setExpMonth(12);
card.setExpYear(2025);
card.setCvn("131");
card.setCardHolderName("Víctor Cantera");
}

// Lanzamos la petición al servidor de Addon Payments
@Test
public void Credit() throws ApiException {

try {
// Procesamos un ingreso
Transaction response = card.refund(new BigDecimal("19.19"))
.withCurrency("EUR")
.execute();

// Obtenemos la respuesta y la mostramos
String result = "Resultado == " + response.getResponseCode(); // 00 == Success
String message = "Mensaje == " + response.getResponseMessage(); // [ test system ] AUTHORISED
String orderId = "Order ID == " + response.getOrderId(); // ezJDQjhENTZBLTdCNzNDQw
String authCode = "Código de autorización == " + response.getAuthorizationCode(); // Código de autorización
String paymentsReference = "Transaction ID (PasRef) == " + response.getTransactionId(); // Identificador de transacción (PASREF)
String schemeReferenceData = "Schema Reference == " + response.getSchemeId(); // MMC0F00YE4000000715

} catch (ApiException exc) {
// Añada aquí su tratamiento de errores
String errorMessage = exc.getMessage();
}
}
}

2. Descripciones

2.1 Configuración del terminal

Debemos introducir los datos de configuración que se facilitan en la llamada de cualificación del soporte técnico de Addon Payments.

  • Merchant_ID = Identificador del comercio.
  • SubAccount = Subcuenta faciltiada en la llamada de Subcuenta faciltiada en la llamada de cualificación (si se deja en blanco se envía la subcuenta por defecto que le hayan creado).
  • Shared_Secret = Contraseña de seguridad del terminal.
  • Refund_Password = Contraseña de Refund.
--> En caso de que le falte algún dato de los mencionados arriba, debe contactar con su soporte técnico de Addon Payments llamando al teléfono 914 353 028 (opción 2) o enviando un email al correo electrónico soporteaddonpayments.com.
GatewayConfig config = new GatewayConfig();
config.setMerchantId("addonjavatest");
config.setAccountId("api");
config.setSharedSecret("secret");
config.setRefundPassword("refund");
config.setServiceUrl("https://remote.addonpayments.com/remote"); // Se debe emplear una de las siguientes URL's:
// Sandbox URL para el entorno de pruebas - https://remote.sandbox.addonpayments.com/remote.
// Live URL para el entorno de producción - https://remote.addonpayments.com/remote.
ServicesContainer.configureService(config);
2.2 Datos de la tarjeta

Para crear el objeto "tarjeta" necesitamos los siguientes parámetros:

  • Llamamos a la función new CreditCardData();
  • Indicamos cuál es el número de la tarjeta a la cual vamos a realizar el ingreso: "4263970000005262";
  • Indicamos el mes de expiración: "12";
  • Indicamos la fecha de expiración: "2025";
  • Nombre del titular de la tarjeta: "Nombre del titular";
card = new CreditCardData();
card.setNumber("4263970000005262");
card.setExpMonth(12);
card.setExpYear(2025);
card.setCvn("131");
card.setCardHolderName("Víctor Cantera");
2.3 Envío de la operación

Para enviar la operación debemos emplear el modelo de excepciones "try - catch".

  • Indicamos al objeto "card" que vamos a realizar un reembolso de 19.19€ Transaction response = card.refund(new BigDecimal("19.19"))

  • Posteriormente debemos indicar la moneda en la que vamos a realizar el reembolso: .withCurrency("EUR")

  • Por último, enviamos la petición: .execute();
// Lanzamos la petición al servidor de Addon Payments
@Test
public void Auth() throws ApiException {

try {
// Procesamos un ingreso
Transaction response = card.refund(new BigDecimal("19.19"))
.withCurrency("EUR")
.execute();

// Obtenemos la respuesta y la mostramos
String result = "Resultado == " + response.getResponseCode(); // 00 == Success
String message = "Mensaje == " + response.getResponseMessage(); // [ test system ] AUTHORISED
String orderId = "Order ID == " + response.getOrderId(); // ezJDQjhENTZBLTdCNzNDQw
String authCode = "Código de autorización == " + response.getAuthorizationCode(); // Código de autorización
String paymentsReference = "Transaction ID (PasRef) == " + response.getTransactionId(); // Identificador de transacción (PASREF)
String schemeReferenceData = "Schema Reference == " + response.getSchemeId(); // MMC0F00YE4000000715

} catch (ApiException exc) {
// Añada aquí su tratamiento de errores

}
}

Para la respuesta obtenemos en la variable 'errorMessage' el resultado de la función "getMessage()".

Con esta función obtenemos el motivo de la denegación del servidor. Para ver todos los errores de denegación actualizados, puede comprobar nuestra documentación de Addon Payments.
